如果报错提示无法删除，大概率情况下是指定删除的镜像docker内在运行，直接停止就行。
```bash
docker rm 6026f10530db：删除已停止的容器
docker rm -f 6026f10530db：删除正在运行的容器
```

#### 进入容器执行命令
重要的是`CONTAINER ID`值，执行`docker ps`查看就可以：
```bash
sudo docker exec -it 775c7c9ee1e1 /bin/bash
```
